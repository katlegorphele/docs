## Prerequisites

Your email must be registered as a Kotani Pay integrator before you can authenticate. You can also log in to the dashboard at [backoffice.kotanipay.com](https://backoffice.kotanipay.com) to manage your account, view transactions, and configure settings — the dashboard uses a separate email + password login and is independent from the API authentication flow below.

---

## 1. Get a JWT via Magic Link

The API uses an email-based magic link flow. No password involved.

```bash
curl -X POST https://sandbox-api.kotanipay.com/api/v3/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email": "you@yourcompany.com"}'
```

Response:

```json
{ "message": "login magic link sent to your email" }
```

Check your inbox and click the **Login to Kotani Pay** button. That link hits `GET /api/v3/auth/verify?hash=...` and returns:

```json
{
  "data": {
    "user_id": "abc123",
    "session_id": "sess_xyz",
    "token_id": "tok_abc",
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "refresh_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }
}
```

The `token` expires in **1 hour**. Use [`GET /api/v3/auth/refresh-token`](/v3/api-reference/authentication/refresh-token) with the `refresh_token` to extend your session without re-doing the magic link flow.

---

## 2. Generate Your API Key

Use the JWT to generate an API key. Two options:

**Key only** (no secret):

```bash
curl -X GET https://sandbox-api.kotanipay.com/api/v3/auth/api-key \
  -H "Authorization: Bearer <your-jwt-token>"
```

**Key + secret** (for end-to-end payload verification):

```bash
curl -X GET https://sandbox-api.kotanipay.com/api/v3/auth/api-key/secure \
  -H "Authorization: Bearer <your-jwt-token>"
```

Both return:

```json
{
  "message": "Api-key generated successfully",
  "data": {
    "key": "eyJ1c2VyX2lkIjoiLi4uIn0=.a1b2c3d4e5f6...",
    "secret": "7f3a9b2c1d4e5f6..."
  }
}
```

The `/api-key` endpoint returns only `key`. The `/api-key/secure` endpoint returns both `key` and `secret`. Store them securely. See [API Keys](/v3/getting-started/api-keys) for key types and permissions.

---

## 3. Make Your First Call

Use the API key in the `Authorization: Bearer` header:

```bash
curl -X GET https://sandbox-api.kotanipay.com/api/v3/health \
  -H "Authorization: Bearer <your-api-key>"
```

Expected response:

```json
{
  "status": "ok",
  "timestamp": "2025-01-01T00:00:00Z"
}
```

---

## 4. Fund Your Payout Balance (Before Withdrawals)

Deposits credit your **deposit balance**. Withdrawals draw from your **payout balance**. These are separate — you need to transfer between them before you can disburse.

In the dashboard: **Wallets → Transfer Deposit Balance**

See [Balances & Settlement](/v3/essentials/balances-and-settlement) for the full explanation.

---

## 5. Set Up Webhooks

Configure your webhook URL so Kotani Pay can notify you when transactions update:

1. Log in to the dashboard → **Settings**
2. Enter your HTTPS endpoint URL
3. Select the events you want (or leave all selected)
4. Copy the generated signing secret and save it as an environment variable

Verify every incoming webhook using the `X-Kotani-Signature` header. See [Webhooks](/v3/essentials/webhooks) for the verification code.

---

## 6. Go Live

When you're ready for production:

1. Switch the base URL to `https://api.kotanipay.com`
2. Repeat the magic link login against production to get a production JWT
3. Generate a production API key
4. Update your webhook endpoint to point to your production server

<Warning>
  Always use sandbox credentials against the sandbox URL and production credentials against the production URL. Never mix them.
</Warning>
